### 实验过程
本次实验围绕供应商-零件数据库及学生课程数据库展开，先通过SQL语句创建数据库和数据表（含主外键约束），再使用ALTER语句修改表结构，插入测试数据验证完整性规则，最后用DROP语句删除表，全程关注表创建和删除的顺序以满足参照完整性要求。

### 实验详细操作步骤或程序清单
#### 1. 创建学生课程数据库表
```sql
-- 创建School数据库（若不存在）
CREATE DATABASE IF NOT EXISTS School;
USE School;

-- 创建student表
CREATE TABLE student (
    sno CHAR(8) PRIMARY KEY,
    sname VARCHAR(8),
    ssex CHAR(2),
    sdept VARCHAR(20),
    sage INT
);

-- 创建course表
CREATE TABLE course (
    cno CHAR(3) PRIMARY KEY,
    cname VARCHAR(20),
    Credit INT,
    Cpno CHAR(3),
    FOREIGN KEY (Cpno) REFERENCES course(cno)
);

-- 创建sc表
CREATE TABLE sc (
    sno CHAR(8),
    cno CHAR(3),
    Grade INT,
    PRIMARY KEY (sno, cno),
    FOREIGN KEY (sno) REFERENCES student(sno),
    FOREIGN KEY (cno) REFERENCES course(cno)
);
```

#### 2. 创建供应商-零件数据库及表
```sql
-- 创建供应商-零件数据库
CREATE DATABASE IF NOT EXISTS Supplier_Part;
USE Supplier_Part;

-- 创建供应商表S
CREATE TABLE S (
    Sno CHAR(5) NOT NULL UNIQUE PRIMARY KEY,
    Sname VARCHAR(20) UNIQUE NOT NULL,
    Status INT,
    City VARCHAR(20)
);

-- 创建零件表P
CREATE TABLE P (
    Pno CHAR(5) NOT NULL UNIQUE PRIMARY KEY,
    Pname VARCHAR(20) NOT NULL,
    Color VARCHAR(10),
    Weight DECIMAL(5,1),
    City VARCHAR(20)
);

-- 创建供应表SP
CREATE TABLE SP (
    Sno CHAR(5),
    Pno CHAR(5),
    Qty INT,
    PRIMARY KEY (Sno, Pno),
    FOREIGN KEY (Sno) REFERENCES S(Sno),
    FOREIGN KEY (Pno) REFERENCES P(Pno)
);
```

#### 3. 修改student表结构（切换回School数据库）
```sql
USE School;

-- 1. sno设为非空和唯一（已为主键，默认满足，可补充约束）
ALTER TABLE student MODIFY sno CHAR(8) NOT NULL UNIQUE;

-- 2. 增加sbirth字段（sname后）和address字段
ALTER TABLE student ADD COLUMN sbirth DATETIME AFTER sname;
ALTER TABLE student ADD COLUMN address VARCHAR(50);

-- 3. 删除address字段
ALTER TABLE student DROP COLUMN address;

-- 4. 修改sbirth列名为birthday
ALTER TABLE student RENAME COLUMN sbirth TO birthday;

-- 5. 修改sage位置（birthday后）
ALTER TABLE student MODIFY sage INT AFTER birthday;

-- 6. 修改sname列名为stuname
ALTER TABLE student RENAME COLUMN sname TO stuname;
```

```sql

USE School;

DROP TABLE IF EXISTS course;
DROP TABLE IF EXISTS student;

-- 删除供应商-零件表（先删SP）

DROP TABLE IF EXISTS P;

```

### 疑难小结
1. **表创建顺序错误**：创建含外键的表（如SP、sc）时，需先创建被参照表（S、P或student、course），否则外键约束无法生效。
2. **字段修改位置问题**：使用`AFTER`关键字调整字段位置时，需确保目标字段存在，否则会报错。
3. **数据插入顺序**：插入外键关联数据时，需先插入主表数据（如S、P），再插入从表数据（如SP），避免违反参照完整性。

2. 通过ALTER语句完成student表结构的修改（字段增删、重命名、位置调整），表结构更新正确。
3. 测试数据按顺序插入各表，无完整性冲突，查询数据显示正常。
4. 按“先删从表后删主表”的顺序成功删除所有表，数据库对象清理完毕。
