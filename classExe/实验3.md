### 实验过程
本次实验基于实验一构建的学生-课程数据库，通过编写并执行不同类型的SQL查询语句，系统练习了简单查询、条件筛选、排序、分组统计及聚合函数的使用，验证了各类查询语法的正确性，掌握了SQL数据检索的核心技能。

### 实验详细操作步骤或程序清单
```sql
-- 切换到School数据库
USE School;

-- 1）查询全体学生的学号和姓名
SELECT sno, sname FROM student;

-- 2）查询全体学生的详细记录
SELECT * FROM student;

-- 3）查询所有选修过课程的学生学号（去重）
SELECT DISTINCT sno FROM sc;

-- 4）查询考试有不及格的学生学号（成绩<60）
SELECT DISTINCT sno FROM sc WHERE Grade < 60;

-- 5）查询不是信息系（IS）、计算机系（CS）的学生性别、年龄、系别
-- 方法一：使用NOT IN
SELECT ssex, sage, sdept FROM student WHERE sdept NOT IN ('IS', 'CS');
-- 方法二：使用AND + <>
SELECT ssex, sage, sdept FROM student WHERE sdept <> 'IS' AND sdept <> 'CS';

-- 6）查询选修了4号课的学生学号和成绩，结果按成绩降序排列
SELECT sno, Grade FROM sc WHERE cno = '004' ORDER BY Grade DESC;

-- 7）查询每个课程号和相应的选课人数
SELECT cno, COUNT(sno) AS 选课人数 FROM sc GROUP BY cno;

-- 8）查询计算机系（CS）的学生姓名、年龄、系别
SELECT sname, sage, sdept FROM student WHERE sdept = 'CS';

-- 9）查询年龄18－20岁的学生学号、姓名、系别、年龄
-- 方法一：使用BETWEEN...AND
SELECT sno, sname, sdept, sage FROM student WHERE sage BETWEEN 18 AND 20;
-- 方法二：使用>=和<=
SELECT sno, sname, sdept, sage FROM student WHERE sage >= 18 AND sage <= 20;

-- 10）查询姓刘的学生情况（模糊查询）
SELECT * FROM student WHERE sname LIKE '刘%';

-- 11）查询学生的姓名和出生年份（今年2025年）
SELECT sname, 2025 - sage AS 出生年份 FROM student;

-- 12）查询没有成绩的学生学号和课程号（成绩为NULL）
SELECT sno, cno FROM sc WHERE Grade IS NULL;

-- 13）查询总成绩大于200分的学生学号
SELECT sno FROM sc GROUP BY sno HAVING SUM(Grade) > 200;

-- 14）查询每门课程不及格学生人数（成绩<60）
SELECT cno, COUNT(sno) AS 不及格人数 FROM sc WHERE Grade < 60 GROUP BY cno;

-- 15）查询不及格课程超过3门的学生学号
SELECT sno FROM sc WHERE Grade < 60 GROUP BY sno HAVING COUNT(cno) > 3;

-- 16）查询年龄在10到19岁之间的学生信息
SELECT * FROM student WHERE sage BETWEEN 10 AND 19;

-- 17）查询全体学生情况，按所在系升序排列，同一个系的学生按年龄降序排列
SELECT * FROM student ORDER BY sdept ASC, sage DESC;

-- 18）查询选了1号课程的学生平均成绩
SELECT AVG(Grade) AS 平均成绩 FROM sc WHERE cno = '001';

-- 19）查询选了3号课程的学生的最高分
SELECT MAX(Grade) AS 最高分 FROM sc WHERE cno = '003';

-- 20）查询每个同学的总成绩
SELECT sno, SUM(Grade) AS 总成绩 FROM sc GROUP BY sno;
```

### 疑难小结
1. **NULL值判断错误**：最初尝试用`Grade = NULL`查询无成绩记录，结果为空，后纠正为`Grade IS NULL`，理解了NULL值的特殊判断规则。
2. **分组筛选混淆**：查询总成绩大于200分的学生时，误将`SUM(Grade) > 200`写在`WHERE`子句中，后改为`HAVING`子句，明确了`WHERE`筛选行、`HAVING`筛选分组的区别。
3. **模糊查询通配符使用**：查询姓刘的学生时，误写为`LIKE '%刘%'`（包含“刘”字），后调整为`LIKE '刘%'`（以“刘”开头），精准匹配需求。
4. **去重关键字遗漏**：查询选修过课程的学生学号时，未加`DISTINCT`导致重复学号，添加后结果正确，掌握了去重的应用场景。

### 实验结果
1. 所有SQL查询语句均可正确执行，返回结果与需求完全匹配。
2. 掌握了`SELECT`语句的基本结构，以及`WHERE`条件筛选、`ORDER BY`排序、`GROUP BY`分组、`HAVING`分组筛选和聚合函数（`COUNT`/`SUM`/`AVG`/`MAX`）的使用。
3. 学会了多种等效查询方法（如`NOT IN`与`AND`组合、`BETWEEN`与比较运算符），提升了SQL查询的灵活性。
